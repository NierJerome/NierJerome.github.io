前端自检清单

变量和类型

  1. JavaScript规定了几种数据类型？
    简单类型：string 、number 、boolean 、null 、undefined 、symbol
    复杂类型：Object

  2. JavaScript对象的底层数据结构是什么？
    JavaScript时使用堆（heap）和栈（stack）存储数据。
    JavaScript基本类型数据直接按值存储在栈中的，每种类型的数据占据的内存空间的大小是确定的，由系统自动分配和自动释放。
    优点内存可以及时回收，相较于堆来说更容易管理内存空间。
    JavaScript引用数据类型数据本身存放在堆中，由一个存放在栈中的地址指针引用。

  3. Symbol类型在实际开发中的应用、可手动实现一个简单的Symbol?
    ES5中的对象属性名都是字符串，容易造成属性名冲突，Symbol表示一个独一无二的值，因此适合作为对象属性名来代替字符串属性名。

  4. JavaScript中的变量在内存中的具体存储形式
    变量的储存方式取决于变量的类型是基本类型还是引用类型

    基本类型：以键值对的方式储存在栈内存中
    复杂类型：会在堆内存中开辟一块空间，储存这个对象的值，并同时在栈内存中储存变量和指向对象的指针

  5. 基本类型对应的内置对象，以及他们之间的装箱拆箱操作
    JavaScript中的引用类型含有三个基本包装类型，分别为Boolean 、String 、Number。
    每当读取一个基本类型时，后台会创建一个对应的基本包装类型的对象，便于我们调用一些方法来操纵这些数据。
    如

    在创建基本类型的基本包装类型过程叫做装箱，即基础数据类型转为引用数据类型。
    装箱操作分为显示装箱与隐式装箱
    隐式装箱一般为对基本类型执行引用类型的操作
    显示装箱一般为new操作符创建。

ES5的对象属性名都是字符串，这容易造成属性名的冲突。如果有一种机制，保证每个属性的名字都是独一无二的就好了，这样就从根本上防止属性名的冲突。这就是 ES6 引入Symbol的原因。

关于 NPM 下不同包全局安装与本地安装的区别及意义？

Posted on 2020-10-29

webpack 官网中给出的两种安装方式

什么是全局安装？

安装位置

全局安装方式是键入命令：npm install -g *包名*或 npm install --global *包名*
(-g 的含义是代表安装到全局环境里面）安装完成后会放在 _nodejs/node_modules_ 中
可以使用命令npm root -g查看 nodejs 的安装根目录

调用方式

全局安装后可以供命令行(command line)使用，用户可以在命令行中直接运行该组件包支持的命令

什么是本地安装 ？

安装位置

本地安装方式是键入命令：npm install *包名* 或 npm install --save-dev *包名*(–save-dev 的含义是代表安装到当前目录中）一般我们再项目中使用其它依赖时都需要，通过终端（cmd，powershell 等）转到项目目录下，进行安装。

调用方式

本地安装后可以直接通过 require()的方式引入项目中 node_modules 目录下的模块

全局安装与本地安装的使用方式 ？

如果只是全局安装

  1. 不能直接通过require()的方式去引用模块，需要手动解决包路径的配置问题，当然你也可以复制全局安装的node_modules文件夹到项目下，还有办法可以选择将环境变量的NODE_PATH设置为C:\Program Files\nodejs。设置环境变量请参考…
  2. 对于包的更新不好管理，可能你需要为每个包重新命名，如 包@3.8.1、包@3.9.1… ，为了区别不同项目使用指定的包，保证模块之间的相互依赖，区别每个项目正常运行。

本地安装的特点

由于包的版本在不断更替，特别的当不同包引入了破坏式变更(breaking change)的依赖，则会使项目发生编译出错等bug。
本地安装可以让每个项目拥有独立的包，不受全局包的影响，方便项目的移动、复制、打包等，保证不同版本包之间的相互依赖，这些优点是全局安装难以做到的。这也使得多人协作中不用刻意管理不同的依赖。

另外，据node团队介绍，本地安装包对于项目的加载会更快。

有优点也少不了缺点，如每次新项目都要本地安装所依赖的包，安装包时间相对较长，一来是包太大导致下载慢；二是浪费了硬盘空间。
